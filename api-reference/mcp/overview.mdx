The Memex MCP server is available at:

`https://mcp.memex.garden/mcp`

It exposes two tools:

- `search_content`
- `save_content_by_url`

Both call the same backend handlers as the REST API.

## Tool result format

For Memex MCP tools, successful `tools/call` responses contain:

- `result.content`: text blocks (first block is JSON text)
- `result.structuredContent`: parsed object payload (recommended)

In practice, `structuredContent` is the same object serialized into `content[0].text`.

## Output shape parity

MCP does not define a different Memex payload schema. It wraps the same operation payloads as REST:

- `search_content` => same payload shape as REST `POST /search-content`
- `save_content_by_url` => same payload shape as REST `POST /save-content-by-url`

Canonical contract:

- [Response shapes](/api-reference/response-shapes)

## Authentication

Send either:

- Headers: `x-user-id` and `x-api-key`
- Query params: `userId` and `apiKey`

The API key is validated against the user's MCP settings.

## List tools

```bash
curl -X POST "https://mcp.memex.garden/mcp" \
  -H "Content-Type: application/json" \
  -H "x-user-id: YOUR_USER_ID" \
  -H "x-api-key: YOUR_MCP_API_KEY" \
  -d '{
    "jsonrpc": "2.0",
    "id": "tools-list-1",
    "method": "tools/list",
    "params": {}
  }'
```

## Call a tool

```bash
curl -X POST "https://mcp.memex.garden/mcp" \
  -H "Content-Type: application/json" \
  -H "x-user-id: YOUR_USER_ID" \
  -H "x-api-key: YOUR_MCP_API_KEY" \
  -d '{
    "jsonrpc": "2.0",
    "id": "call-1",
    "method": "tools/call",
    "params": {
      "name": "search_content",
      "arguments": {
        "query": "vector databases",
        "limit": 5
      }
    }
  }'
```

<Warning>
  MCP `search_content` deducts `1` credit per request, plus `+1` credit if rerank is enabled. Direct REST calls do not apply MCP billing unless internal flags are explicitly enabled.
</Warning>
