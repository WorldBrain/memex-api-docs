## Response format

The endpoint returns:

- `entity: ContentEntity`
- `tags: TagEntity[]`
- `isNew: boolean`

## Cross-interface parity

This save response shape is identical across:

- REST `POST /save-content-by-url`
- MCP `save_content_by_url` (inside `result.structuredContent`)
- OpenClaw skill calls that proxy this endpoint

The `entity` object is the same `ContentEntity` schema used by search results.

### Field behavior

| Field | Type | Meaning |
| --- | --- | --- |
| `entity` | object | The saved or updated library entry hydrated from storage. |
| `tags` | array | Tags resolved from this request (existing + newly created). |
| `isNew` | boolean | `true` when a new entry is created, `false` when an existing entry is reused/updated. |

## `entity` variants

`entity` uses the same `ContentEntity` union as search responses.

- Web: `type: "web"`
- Tweet: `type: "twitter"`
- YouTube: `type: "youtube"`
- Image: `type: "image"`
- Reddit: `type: "reddit"` (documented for compatibility; public reddit URL ingestion is currently unsupported)

For full per-type metadata fields, see:

- [Response shapes](/api-reference/response-shapes)

```json
{
  "entity": {
    "id": "98cc2351-920e-4948-b54c-c359ecc4624e",
    "type": "web",
    "url": "https://example.com/article",
    "title": "Example article"
  },
  "tags": [
    {
      "id": "2df87b47-2c89-4a3d-8c2a-566ea3adfe95",
      "name": "reading"
    },
    {
      "id": "a5a0bca8-3e5e-4f47-a1ee-f0ef9a84b2fd",
      "name": "research"
    }
  ],
  "isNew": true
}
```
