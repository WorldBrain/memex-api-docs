Use this page to add a Memex skill in OpenClaw format.

This page intentionally does not include OpenClaw installation steps.

## 1) Create the skill file

Preferred (workspace-only):

`<your-workspace>/skills/memex-api/SKILL.md`

Shared across all workspaces on your machine:

`~/.openclaw/skills/memex-api/SKILL.md`

Copy and paste this file content:

```md
---
name: memex-api
description: Search and save content in Memex via REST API.
metadata: { "openclaw": { "primaryEnv": "MEMEX_API_KEY", "requires": { "env": ["MEMEX_API_KEY"] } } }
---

Use this skill when users ask to search or save content in Memex.

Search endpoint:
- POST https://api.memex.garden/search-content
- Headers:
  - Authorization: Bearer ${MEMEX_API_KEY}
  - Content-Type: application/json

Save endpoint:
- POST https://api.memex.garden/save-content-by-url
- Headers:
  - Authorization: Bearer ${MEMEX_API_KEY}
  - Content-Type: application/json
```

## 2) Put the API key secret in OpenClaw config

Set the secret in:

`~/.openclaw/openclaw.json`

```json5
{
  skills: {
    entries: {
      "memex-api": {
        enabled: true,
        apiKey: { source: "env", provider: "default", id: "MEMEX_API_KEY" }
      }
    }
  }
}
```

Alternative (inline value in config):

```json5
{
  skills: {
    entries: {
      "memex-api": {
        enabled: true,
        env: {
          MEMEX_API_KEY: "YOUR_MEMEX_API_KEY"
        }
      }
    }
  }
}
```

`metadata.openclaw.primaryEnv` must match the env var you use for the key.

## 3) Reload skills

Start a new OpenClaw session after changing `SKILL.md` or `openclaw.json`.

## 4) Return the Memex payload unchanged

When this skill calls Memex endpoints, return the JSON response object as-is.

- For `search-content`, keep the full search payload shape unchanged.
- For `save-content-by-url`, keep `{ entity, tags, isNew }` unchanged.

Canonical contract:

- [Response shapes](/api-reference/response-shapes)
